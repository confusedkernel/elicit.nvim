elicit.nvim                                               *elicit.nvim*

Elicit is a Neovim plugin for linguistic fieldwork.
Session files are Markdown with YAML frontmatter.

This help file documents the current v0.1 MVP progress.

==============================================================================
1. Setup                                                       *elicit-setup*

Initialize from your config:

>
  require("elicit").setup({
    session = {
      dir = "sessions",
    },
  })
<

If setup is not called, defaults are used.

==============================================================================
2. Commands                                                 *elicit-commands*

                                                        *:ElicitNewSession*
:ElicitNewSession [name]
  Create a new session file under `session.dir`.
  If [name] is omitted, a default name is generated.
  If [name] has no `.md` extension, `.md` is appended.

                                                        *:ElicitNewExample*
:ElicitNewExample
  Insert a new elicitation example block at the cursor location.
  Example ID is generated from language, date, and session-local counter.

                                                          *:ElicitValidate*
:ElicitValidate
  Validate current session and populate quickfix diagnostics.
  Checks Segmentation/Gloss token alignment, empty required fields,
  and placeholder markers configured in `validation.placeholders`.

                                                            *:ElicitSearch*
:ElicitSearch {kind} {query}
  Search the corpus by kind and query.
  Kinds: form, gloss, status, speaker, session.
  Results open in the configured backend (`search.backend`).
  If backend is `telescope` but telescope is unavailable,
  results fall back to quickfix.

                                                            *:ElicitExport*
:ElicitExport {format} {scope} [value]
  Export data in selected format.
  Formats: markdown, typst, json.
  Scopes: example, session, status.
  (Planned for phase 5.)

==============================================================================
3. Configuration                                               *elicit-config*

Default config:

>
  require("elicit").setup({
    session = {
      dir = "sessions",
      fields = {
        "date", "language", "speaker", "session",
        "location", "topic", "audio", "status", "tags",
      },
      defaults = {
        language = "",
        status = "in-progress",
      },
    },
    example = {
      id_format = "LID-YYYYMMDD-NNN",
      fields = {
        "Prompt", "Text", "Segmentation", "Gloss",
        "Translation", "Notes", "Status", "Audio",
      },
      required_fields = { "Text", "Translation" },
      status_cycle = { "draft", "needs-review", "checked" },
    },
    validation = {
      delimiter = "%s+",
      placeholders = { "?", "TODO", "XXX" },
    },
    search = {
      backend = "telescope", -- or "quickfix"
      corpus_glob = nil,
    },
    export = {
      formats = { "markdown", "typst", "json" },
      output_dir = "export",
    },
  })
<

When `search.corpus_glob` is nil, corpus discovery uses:
>
  session.dir .. "/**/*.md"
<

==============================================================================
4. Telescope integration                                   *elicit-telescope*

Load the extension after telescope setup:

>
  require("telescope").load_extension("elicit")
<

Then run:

  :Telescope elicit
    Prompt for kind and query, then open matches in Telescope.

  :ElicitSearch {kind} {query}
    Run the same search engine, using configured backend display.
